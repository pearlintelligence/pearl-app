# Security Policy

## Reporting a Vulnerability

We take the security of InnerPearl seriously. If you discover a security
vulnerability, please report it responsibly.

### How to Report

**Email:** [security@innerpearl.ai](mailto:security@innerpearl.ai) or
[shiv@innerpearl.ai](mailto:shiv@innerpearl.ai)

Please include:

- A description of the vulnerability
- Steps to reproduce (if applicable)
- The potential impact
- Any suggested fixes (optional)

### What to Expect

- **Acknowledgement** within 48 hours of your report
- **Status update** within 5 business days
- **Resolution timeline** shared once we've assessed severity

### Responsible Disclosure

- Please give us reasonable time to address the issue before disclosing it
  publicly.
- Do not access, modify, or delete other users' data.
- Do not perform actions that could degrade or disrupt the service.

### Scope

The following are in scope:

- `app.innerpearl.ai` — Pearl web application
- `innerpearl.ai` — Marketing website
- The public GitHub repositories for this project

### Out of Scope

- Denial-of-service attacks
- Social engineering / phishing
- Issues in third-party dependencies (report upstream; let us know too)

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | ✅        |

We only support the latest deployed version. Please ensure you're testing
against the current production release.
